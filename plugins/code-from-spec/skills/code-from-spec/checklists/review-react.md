# 코드 리뷰 체크리스트 — React (TypeScript)

프로젝트 유형이 React일 때 코드 리뷰 루프에서 사용하는 기본 체크리스트.
프로젝트의 CLAUDE.md, AGENTS.md에 추가 규칙이 있으면 병합한다.

---

## R-01: 컴포넌트 구조

- [ ] 컴포넌트가 단일 책임 원칙을 지키는가
- [ ] 컨테이너/프레젠테이션 분리가 적절한가
- [ ] 컴포넌트 depth가 과도하지 않은가 (3단계 이내 권장)
- [ ] 공통 컴포넌트가 재사용 가능하게 설계되었는가

## R-02: 상태 관리

- [ ] 불필요한 리렌더링이 발생하지 않는가
- [ ] useMemo/useCallback이 적절히 사용되었는가 (과도한 최적화 아닌지)
- [ ] 전역 상태와 로컬 상태가 적절히 분리되었는가
- [ ] 서버 상태 관리(React Query 등)가 올바르게 사용되었는가

## R-03: 타입 안전성

- [ ] any 타입 사용이 없는가
- [ ] API 응답 타입이 명확히 정의되었는가
- [ ] Props 타입에 optional이 남용되지 않았는가
- [ ] 유니온/인터섹션 타입이 적절히 사용되었는가

## R-04: 에러 핸들링

- [ ] API 호출에 에러 핸들링이 있는가
- [ ] ErrorBoundary가 적절히 배치되었는가
- [ ] 사용자에게 에러 상태를 적절히 보여주는가
- [ ] 로딩/에러/빈 상태가 모두 처리되었는가

## R-05: 보안

- [ ] XSS 취약점이 없는가 (dangerouslySetInnerHTML 사용 확인)
- [ ] 민감 정보가 클라이언트에 노출되지 않는가
- [ ] 인증 토큰이 안전하게 관리되는가
- [ ] 외부 입력값이 적절히 sanitize 되는가

## R-06: 성능

- [ ] 대용량 리스트에 가상화(virtualization)가 적용되었는가
- [ ] 이미지 최적화(lazy loading, 적절한 크기)가 되어 있는가
- [ ] 번들 크기에 영향을 주는 불필요한 의존성이 없는가
- [ ] 코드 스플리팅이 적절히 적용되었는가

## R-07: 테스트

- [ ] 핵심 비즈니스 로직에 단위 테스트가 있는가
- [ ] 사용자 인터랙션 테스트(RTL)가 있는가
- [ ] Mock이 과도하지 않은가
- [ ] 경계값/예외 케이스 테스트가 있는가

## R-08: 코드 스타일

- [ ] 미사용 import/변수가 없는가
- [ ] ESLint/Prettier 규칙을 따르는가
- [ ] 매직 넘버/문자열이 상수로 추출되었는가
- [ ] 네이밍이 일관적인가 (컴포넌트: PascalCase, 훅: useCamelCase)

## R-09: 접근성

- [ ] 시맨틱 HTML 태그를 사용하는가
- [ ] aria 속성이 적절히 사용되었는가
- [ ] 키보드 네비게이션이 가능한가
- [ ] 색상 대비가 충분한가
