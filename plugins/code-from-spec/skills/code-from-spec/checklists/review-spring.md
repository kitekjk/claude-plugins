# 코드 리뷰 체크리스트 — Spring (Java/Kotlin)

프로젝트 유형이 Spring일 때 코드 리뷰 루프에서 사용하는 기본 체크리스트.
프로젝트의 CLAUDE.md, AGENTS.md, `.github/workflows/claude-review.yml`에 추가 규칙이 있으면 병합한다.

---

## R-01: 모듈 의존성 방향

- [ ] 의존성이 단방향인가 (app → core → common)
- [ ] core 모듈이 app 모듈을 참조하지 않는가
- [ ] 순환 의존성이 없는가

## R-02: 레이어 분리

- [ ] @UseCase는 오케스트레이션만 수행하는가 (직접 DB 접근 금지)
- [ ] @Service는 단일 도메인 로직만 담당하는가
- [ ] @Transactional이 Service/UseCase 메서드 레벨에만 있는가
- [ ] 엔티티/VO를 API에 직접 노출하지 않고 DTO(record)로 변환하는가
- [ ] Controller → UseCase/Service → Repository 계층을 지키는가

## R-03: JPA / 데이터 접근

- [ ] N+1 문제가 없는가 (FetchJoin/EntityGraph 사용 확인)
- [ ] 단순 조회에 Querydsl DTO 프로젝션을 사용하는가
- [ ] @Enumerated(EnumType.STRING) 사용 시 enum 이름 변경 영향을 고려했는가
- [ ] 벌크 연산 시 영속성 컨텍스트 초기화를 수행하는가
- [ ] 인덱스가 필요한 쿼리에 적절한 인덱스가 있는가

## R-04: 에러 핸들링

- [ ] 도메인별 커스텀 예외를 사용하는가 (NotFoundXxxError 패턴)
- [ ] 예외 메시지가 디버깅에 충분한 정보를 포함하는가
- [ ] checked exception을 무시하거나 빈 catch 블록이 없는가
- [ ] 외부 시스템 호출에 적절한 fallback/retry가 있는가

## R-05: 보안

- [ ] API 키/시크릿이 하드코딩되어 있지 않은가
- [ ] 로그에 민감정보(토큰, 개인정보)가 노출되지 않는가
- [ ] 외부 시스템 credential이 환경변수/시크릿 관리되는가
- [ ] SQL 인젝션 취약점이 없는가 (네이티브 쿼리 사용 시)
- [ ] 인증/인가 체크가 누락된 엔드포인트가 없는가

## R-06: 성능

- [ ] 불필요한 DB 조회가 반복되지 않는가
- [ ] 캐시 적용이 필요한 마스터 데이터 조회에 캐시가 있는가
- [ ] 데이터 수정 시 관련 캐시 무효화가 빠지지 않았는가
- [ ] 대량 데이터 처리 시 배치/페이징을 사용하는가
- [ ] Connection Pool 설정이 적절한가

## R-07: 테스트

- [ ] 새로운 로직에 대응하는 테스트가 있는가
- [ ] 테스트가 @Nested 구조로 정리되어 있는가
- [ ] 테스트 메서드명이 한글로 명확한가
- [ ] Mock 사용이 과도하지 않은가 (통합 테스트 우선)
- [ ] 경계값/예외 케이스 테스트가 있는가

## R-08: 코드 스타일

- [ ] @Data, @Setter 어노테이션을 사용하지 않는가
- [ ] DTO에 record 타입 + @Builder를 사용하는가
- [ ] final 키워드가 적절히 사용되었는가
- [ ] 미사용 import/변수가 없는가
- [ ] Spotless 포맷팅이 적용되었는가

## R-09: 비동기 / 이벤트

- [ ] Kafka Consumer에 멱등성 처리가 있는가
- [ ] 재시도 대상 예외가 올바르게 구분되는가
- [ ] Dead Letter Queue 핸들링이 있는가
- [ ] 트랜잭션 경계가 올바른가 (이벤트 발행 시점)

## R-10: 감사 / 추적

- [ ] @Audited 엔티티 변경 시 감사 테이블 부하를 고려했는가
- [ ] 대량 데이터 변경이 감사 로그 폭발을 유발하지 않는가
