# 검증 완료 체크리스트

## 영역별 검증

### V-01: 기능 구현 완결성 (40점)
- [ ] Use Case Spec에서 UC-ID 목록 추출
- [ ] 각 UC-ID에 대응하는 Application Service 확인
- [ ] API Spec에서 엔드포인트 목록 추출
- [ ] 각 엔드포인트에 대응하는 Controller 메서드 확인
- [ ] 요청/응답 필드 Spec 일치 확인
- [ ] 에러 응답 처리 확인
- [ ] 누락/불일치 항목별 상세 기록

### V-02: 비즈니스 규칙 준수 (30점)
- [ ] policies/에서 비즈니스 규칙 목록 추출
- [ ] 각 규칙에 대응하는 코드 위치 확인
- [ ] 경계값 처리 검증 (반올림, 절사 등)
- [ ] 상태 전이 규칙 검증
- [ ] 권한/접근 제어 규칙 검증
- [ ] 미준수 항목별 상세 기록

### V-03: 아키텍처 준수 (15점)
- [ ] 레이어 분리 확인 (도메인에 프레임워크 의존성 없음)
- [ ] 의존성 방향 규칙 확인
- [ ] 모듈/패키지 구조 확인
- [ ] 아키텍처 패턴 적용 확인
- [ ] 위반 항목별 상세 기록

### V-04: 테스트 커버리지 (15점)
- [ ] Spec의 모든 TC-ID가 @Tag로 존재
- [ ] 전체 테스트 실행 + 통과 확인
- [ ] Given-When-Then 시나리오 반영 확인
- [ ] 누락 TC-ID 목록 기록
