## 1. 스타일 준수 (Style Compliance)

기본 배점: 40점 (ADR/HLD/LLD는 타입별 체크리스트 기준으로 조정)

### 1.1 Voice & Tone (15점)

| ID | 체크 항목 | 배점 | 평가 기준 | 감점 사유 예시 |
|----|----------|------|----------|---------------|
| S-01 | 전문적+친근한 톤 | 3점 | 권위적이지 않으면서 신뢰감 있는 서술 | "~해야만 한다" 같은 강압적 표현 |
| S-02 | 객관적 서술 | 3점 | 감정적 표현 배제, 사실 기반 | "정말 좋은", "최고의" 같은 주관적 표현 |
| S-03 | 3인칭/수동태 일관성 | 3점 | 기술 문서에 맞는 관점 유지 | "내가 생각하기에" 같은 1인칭 사용 |
| S-04 | 낙관적/성장 지향적 | 3점 | 문제보다 해결책 중심 서술 | 부정적 표현 과다 |
| S-14 | **데이터 중심 서술** | 3점 | 주장은 수치나 사례로 뒷받침 | 근거 없는 "효율적", "빠른" 사용 |

### 1.2 Phrase Bank 준수 (17점)

| ID | 체크 항목 | 배점 | 평가 기준 | 감점 사유 예시 |
|----|----------|------|----------|---------------|
| S-05 | 선호 동사 사용 | 3점 | 구현/설계/검증/분석/확장/도입/채택/선정 등 | "하다", "되다" 반복 사용 |
| S-06 | 적절한 연결어 | 2점 | 따라서/그러나/예를 들어/특히 등 사용 | 연결어 없이 문장 나열 |
| S-07 | 수치 표현 형식 | 4점 | P99 Xms, X TPS, 99.X% 형식 준수 | "빠른 응답", "높은 처리량" |
| S-08 | 기피 표현 회피 | 2점 | 모호한 표현 대신 구체적 대안 사용 | "적절한", "상당한", "효율적인" 단독 사용 |
| S-15 | **결정 표현 사용** | 3점 | "~를 채택합니다", "~를 선정합니다" 형식 | "A로 한다", "A를 쓴다" |
| S-16 | **출처 명시** | 3점 | 모든 수치에 출처 포함 (내부 메트릭/공식 문서/벤치마크) | 출처 없는 성능 수치 사용 |

### 1.3 Formatting (14점)

| ID | 체크 항목 | 배점 | 평가 기준 | 감점 사유 예시 |
|----|----------|------|----------|---------------|
| S-09 | 문단 구성 | 3점 | 한 문단 = 하나의 핵심 메시지, 3-5문장 | 10문장 이상의 긴 문단 |
| S-10 | 제목 레벨 적절성 | 3점 | H1-H4 적절한 계층 구조 | H2 바로 다음에 H4 사용 |
| S-11 | 목록 사용 | 2점 | 순서/비순서 목록 적절 사용 | 나열 항목을 문장으로만 작성 |
| S-12 | 마크다운 형식 | 3점 | 코드 블록, 굵은 글씨, 표 적절 사용 | 코드에 백틱 미사용, 강조 미사용 |
| S-13 | 짧은 문장 | 3점 | 한 문장 한 가지 내용, 40자 이내 권장 | 80자 이상의 복문 |

### 1.4 Persona 적용 (12점)

> Staff Engineer Persona (`memory/personas/staff-engineer.md`) 적용 여부를 평가합니다.
> 문서 타입별로 가중치가 다르게 적용됩니다.

| ID | 체크 항목 | ADR 배점 | HLD 배점 | LLD 배점 | 평가 기준 | 감점 사유 예시 |
|----|----------|---------|---------|---------|----------|---------------|
| P-01 | 시스템 사고 | 2점 | 4점 | **6점** | 상하류 영향, ADR 간 연결, 전체 아키텍처 관점 포함 | 단일 컴포넌트 관점만 존재, 시스템 영향 분석 누락 |
| P-02 | 2단계 근거 | **5점** | 4점 | 3점 | 주요 결정에 "왜?"를 2단계 깊이로 설명 | "성능 때문"만 기술 (1단계), 비즈니스 맥락 누락 |
| P-03 | 의사결정 프레이밍 | **5점** | 4점 | 3점 | 제약 조건, 평가 기준, Do Nothing 옵션 검토 | 옵션 나열만 존재, Do Nothing 비용 미검토 |

**문서 타입별 배점 근거**:
- **ADR**: 개별 결정의 "왜?"가 핵심 → P-02, P-03 가중치 높음
- **HLD**: 시스템 구조와 핵심 설계를 연결해야 하므로 3개 축을 균형 있게 평가
- **LLD**: 시스템 전체 영향과 구현 세부를 함께 봐야 하므로 P-01 가중치 높음

## 2. 명확성 & 흐름 (Clarity & Flow)

기본 배점: 20점 (HLD: 21점, LLD: 21점으로 조정)

| ID | 체크 항목 | 배점 | 평가 기준 | 감점 사유 예시 |
|----|----------|------|----------|---------------|
| C-01 | 논리적 연결성 | 4점 | 섹션 간 자연스러운 흐름 | 갑작스러운 주제 전환 |
| C-02 | 근거 제시 | 4점 | 모든 주장에 이유 포함, "왜?"에 답변 | "A를 선택한다" (근거 없음) |
| C-03 | 예시 포함 | 3점 | 추상적 개념에 구체적 사례 | 개념 설명만 있고 예시 없음 |

| C-05 | 문장 완결성 | 3점 | 모호한 결론 없음, 명확한 서술 | "등을 고려해야 한다" (무엇을?) |
| C-06 | **트레이드오프 명시** | 3점 | 선택의 장단점 모두 기술, 감수하는 단점 인정 | 장점만 나열, 단점 누락 |

| ID | 체크 항목 | 배점 | 평가 기준 | 감점 사유 예시 |
|----|----------|------|----------|---------------|
| O-01 | 균형 있는 비교 | 3점 | 모든 옵션에 장단점 기술 | 채택 옵션 장점만, 탈락 옵션 단점만 나열 |

## 3. 금지표현 위반 (Forbidden Expressions)

기본 배점: 10점 (위반 시 감점, 최소 0점)

### 3.1 절대 금지 (치명적 감점)

| ID | 금지 표현 | 감점 | 대안 |
|----|----------|------|------|
| F-01 | "think outside the box" | -3점 | 삭제 또는 구체적 표현으로 대체 |
| F-02 | "on the same page" | -3점 | "합의된", "동일한 이해" |
| F-03 | "game changer" | -3점 | 구체적 영향 수치로 대체 |
| F-04 | "시너지" | -2점 | "상호 보완 효과", 구체적 결과 명시 |
| F-05 | "혁신적인" (근거 없이) | -2점 | 근거와 함께 사용하거나 삭제 |
| F-06 | "획기적인" (근거 없이) | -2점 | 근거와 함께 사용하거나 삭제 |
| F-07 | "당연히", "자명하게" | -2점 | 근거 제시 후 결론 도출 |

### 3.2 주의 표현 (경고, 감점 없음)

| ID | 표현 | 권장 조치 | 비고 |
|----|------|----------|------|
| F-08 | "기본적으로" | 삭제 또는 구체화 | 의미 없는 수식어 |
| F-09 | "일반적으로" | 예외 명시 필요 | 모호함 유발 |
| F-10 | "등" 남발 | 열거 후에만 사용 | 3회 이상 사용 시 경고 |
| F-11 | "최적의" | 비교 대상과 기준 명시 | 무엇과 비교해서 최적인지 |
| F-12 | "충분히" | 수치로 대체 권장 | "충분히 빠른" → "P99 50ms" |
| F-13 | "적절한" | 구체적 기준 제시 | "적절한 수준" → 수치화 |

### 3.3 출처 없이 사용 금지 (신규 - 감점)

| ID | 항목 | 감점 | 필요한 출처 예시 |
|----|------|------|-----------------|
| F-14 | 성능 수치 (Latency, TPS) | -2점 | 내부 메트릭, 벤치마크 결과 |
| F-15 | 비용 데이터 | -2점 | AWS 가격표, 내부 비용 산정 |
| F-16 | 벤치마크 결과 | -2점 | 테스트 환경, 측정 방법 |
| F-17 | 외부 시스템 SLA/SLO | -2점 | 공식 문서, 계약서 |

## 4. 점수 계산 방법

### 4.1 스타일 준수 점수
```
스타일_점수 = S-01 ~ S-16 합계
최대 46점 (문서 타입별 실제 배점은 타입별 체크리스트 기준)
```

### 4.2 Persona 적용 점수 (신규)
```
Persona_점수 = P-01 + P-02 + P-03
최대 12점 (문서 타입별 가중치 적용)

ADR: P-01(2점) + P-02(5점) + P-03(5점) = 12점
LLD: P-01(6점) + P-02(3점) + P-03(3점) = 12점
```

### 4.3 명확성 & 흐름 점수
```
명확성_점수 = C-01 ~ C-06 합계
최대 20점 (실제 배점은 타입별 체크리스트 기준)
```

### 4.4 옵션 비교 점수 (해당 시)
```
옵션비교_점수 = O-01 + O-02 + O-03 + O-04
최대 10점 (옵션 비교가 있는 문서에만 적용)
```

### 4.5 금지표현 점수
```
금지표현_점수 = 10 - (위반 감점 합계)
최소 0점
- 절대 금지 위반: F-01 ~ F-07
- 출처 미명시 위반: F-14 ~ F-17
```

### 4.6 항목별 최소 점수 기준
각 항목은 최대 배점의 **50% 이상**을 획득해야 합니다.
- 스타일 준수: 실제 최소 점수는 타입별 체크리스트 기준
- **Persona 적용: 최소 6점/12점**
- 명확성 & 흐름: 실제 최소 점수는 타입별 체크리스트 기준
- 옵션 비교: 최소 5점/10점 (해당 시)
- 금지표현: **절대 금지 위반 0건** 필수

## 5. 수정 예시 형식

피드백 시 다음 형식을 사용합니다:

```markdown
**[{체크리스트 ID}] {문제 요약}**
- **위치**: {섹션명} > {문단 또는 라인}
- **현재**: > {문제가 있는 원문}
- **수정**: > {수정된 문장}
- **이유**: {style_guide.md 또는 체크리스트 기준 인용}
```

**예시**: `[S-07] 모호한 수치` - 위치: NFR > Performance - 현재: "빠른 응답 시간" → 수정: "P99 50ms 이하" - 이유: 구체적 수치 필수

## 6. 회차별 평가 템플릿

> 타입별 체크리스트에서 이 템플릿을 참조합니다.

### 1회차: 구조 + 금지표현
```markdown
## 1회차 평가 결과 ({TYPE})
### 치명적 문제 (Must Fix)
- [{ID}] {문제} → {조치}
### 구조 체크
| 섹션 | 상태 |
|------|------|
| {섹션명} | OK / MISSING |
```

### 2회차: 스타일 + 명확성
- S-01~S-16, C-01~C-06, O-01~O-04 (해당 시)

### 3회차: 완성도 + 제약조건
- 총점 88% 확인, 제약조건 (A/H/L-10~13)
