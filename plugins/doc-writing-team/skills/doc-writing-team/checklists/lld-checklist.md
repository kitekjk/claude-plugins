## 3.5 HLD 정합성 (조건부) - 4점

> **연관 HLD가 있을 때만 적용합니다.** HLD 없이 단독 LLD 작성 시 이 섹션의 4점은 다른 항목에 균등 배분합니다.

| ID | 체크 항목 | 배점 |
|----|----------|------|
| L-30 | 연관 HLD 섹션 포함 (HLD 경로, 제목, 대상 컴포넌트) | 1점 |
| L-31 | KDD 추적 — HLD KDD가 LLD 설계 판단 근거에 매핑 | 1점 |
| L-32 | NFR 추적 — HLD NFR이 LLD에서 구체화 | 1점 |
| L-33 | 용어 일관성 — HLD Glossary와 동일 용어 동일 정의 | 1점 |

### 검증 방법

- **L-30**: "연관 HLD" 섹션이 존재하고 HLD 경로가 유효한지 확인
- **L-31**: HLD의 각 KDD가 LLD 설계 판단 근거의 최소 1개 항목과 매핑되는지 확인. 불일치 시 변경 사유가 명시되어 있는지 확인
- **L-32**: HLD NFR의 상위 목표가 LLD에서 API/컴포넌트별 구체 수치로 분해되었는지 확인
- **L-33**: HLD Glossary의 용어가 LLD에서 다른 의미로 사용되지 않는지 확인

## 5. LLD 섹션별 평가 기준

| 섹션 | 우수 (만점) | 보통 (60%) | 미흡 (20%) |
|------|------------|-----------|-----------|
| **Glossary** | 5-15개 용어, 일관된 포맷 | 5개 미만, 불완전 | 섹션 없음 |
| **Problem** | 데이터특성+난제+영향도+수치+리스크 | 영향도 부족, 수치 없음 | "개선 필요" 수준 |
| **NFR** | Performance+Availability 수치 | 1-2개, 한 섹션만 | 모호한 표현만 |
| **Design** | 다이어그램+역할+흐름+근거+API+스키마 | 시각화 없음, API/스키마 부족 | 개념만 |
| **API 설계** | 전체 엔드포인트+요청/응답+에러코드 | 엔드포인트만, 스키마 부족 | 없음 |
| **클래스 설계** | 클래스+메서드+관계+다이어그램 | 클래스 목록만 | 없음 |
| **DB 스키마** | DDL+컬럼 설명+인덱스 | 테이블 목록만 | 없음 |
| **설계 판단** | 채택/탈락 비교+근거+감수하는 단점 | 결정만, 비교 부족 | 없음 |
| **미해결 리스크** | 리스크+영향도+확률+완화 방안 | 리스크만 나열 | 없음 |
