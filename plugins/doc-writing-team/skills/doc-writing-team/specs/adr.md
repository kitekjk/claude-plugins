# ADR 문서 스펙

`ADR`은 아키텍처 또는 기술 의사결정의 맥락, 옵션 비교, 결정 상태, 후속 행동을 기록하는 문서입니다.

## 계약 기준

- 구조와 저장 규칙의 단일 기준은 `skills/doc-writing-team/contract.json`
- 저장 경로는 `documents/adr/{slug}/DOCUMENT.md`
- `Decision`은 작성자가 임의 확정하지 않으며 `[TBD]` 또는 사용자/팀 확인된 선택만 허용
- Appendix는 기본 구조에 포함하지 않음

## 필수 섹션

### 1. Context

- 배경
- 문제 정의
- 필요하면 데이터 특성 또는 성능/비용/운영 제약
- 도입부 변형 2~3개 권장

### 2. Agenda

- 결정해야 할 항목
- 비교 옵션 2개 이상
- 옵션별 장점/단점
- 평가 기준 또는 탈락 사유 태그

### 3. Decision

- 현재 상태: `[TBD]` 또는 사용자/팀 확인된 선택
- 선택 근거
- 트레이드오프
- 미결정 항목

### 4. Next Step

- 실행 단계
- 담당 또는 후속 검증 방법
- 성공 기준

## 작성 규칙

- 최소 2개 이상의 옵션을 비교합니다.
- 수치에는 출처를 남깁니다.
- 탈락 옵션에는 `[Physical Limit]`, `[Operational Risk]`, `[Cost Prohibitive]`, `[Compatibility]` 중 하나를 우선 사용합니다.
- `Decision`에서 writer의 임의 확정 문장을 사용하지 않습니다.
- 구조 외 보조 설명이 필요해도 Appendix를 추가하지 않습니다.

## 권장 길이

| 섹션 | 권장 길이 |
|------|----------|
| Context | 200-400자 |
| Agenda | 300-600자 |
| Decision | 200-400자 |
| Next Step | 100-200자 |
| 전체 | 800-1600자 |
