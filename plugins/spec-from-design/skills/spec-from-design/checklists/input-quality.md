# 입력 품질 체크리스트

HLD/LLD가 Spec 생성에 충분한 정보를 담고 있는지 검증합니다.
필수 항목이 누락되면 Spec 생성을 중단하고 사용자에게 리포트합니다.

## HLD 필수 항목

| # | 항목 | 검증 기준 | 누락 시 영향 |
|---|------|----------|-------------|
| H-01 | **Glossary** | 도메인 용어 정의 3개 이상 | Spec에서 용어 혼동 |
| H-02 | **Overview (배경 + 목적)** | 왜 이 기능이 필요한지 명시 | Use Case 개요 작성 불가 |
| H-03 | **System Context** | 외부 시스템 연동 다이어그램 또는 테이블 | API Spec 연동 대상 불명확 |
| H-04 | **상태 전이 모델** (해당 시) | 상태 enum + 전이 조건 정의 | 분기 커버리지 테스트 누락 |
| H-05 | **핵심 컴포넌트 책임** | 컴포넌트별 역할 테이블 | Use Case 흐름 분리 기준 불명확 |
| H-06 | **KDD** | 결정 + 대안 비교 + 근거 1개 이상 | 정책 Spec 도출 불가 |
| H-07 | **NFR** | 성능/가용성 지표 + 목표 수치 | NFR 테스트 작성 불가 |

## LLD 필수 항목

| # | 항목 | 검증 기준 | 누락 시 영향 |
|---|------|----------|-------------|
| L-01 | **Problem Statement** | 배경 + 목표 + 데이터 특성 | Use Case 개요 부실 |
| L-02 | **Functional Requirements** | FR-ID + 요구사항 + 검증 기준 | 테스트 시나리오 생성 불가 |
| L-03 | **Non-Functional Requirements** | NFR-ID + 카테고리 + 목표 + 측정방법 | NFR 테스트 누락 |
| L-04 | **Goal / Non-Goal** | 구현 범위 명확화 | 불필요한 Spec 생성 위험 |
| L-05 | **API 설계** | 엔드포인트 + 요청/응답 스키마 | API Spec 작성 불가 |
| L-06 | **클래스/컴포넌트 설계** | 주요 클래스 + 메서드 시그니처 | Use Case 흐름 상세화 불가 |
| L-07 | **DB 스키마** (해당 시) | 테이블 + 컬럼 + 인덱스 | 검증 조건 도출 불가 |
| L-08 | **설계 판단 근거** | 옵션 비교 + 감수하는 단점 | 정책 Spec 근거 부족 |
| L-09 | **미해결 리스크** | 리스크 + 영향도 + 완화 방안 | 엣지 케이스 테스트 누락 |

## 선택 항목

| # | 항목 | 있으면 좋은 이유 |
|---|------|----------------|
| O-01 | 사용자 스토리/페르소나 | Use Case Actor 명확화 |
| O-02 | 에러 코드 카탈로그 | API Spec 에러 응답 통일 |
| O-03 | 경계값 정의 | 테스트 엣지 케이스 풍부화 |
| O-04 | 동시성 시나리오 | Integration 테스트 커버리지 향상 |
| O-05 | 배포 전략 / 롤백 계획 | 운영 NFR 테스트 근거 |
| O-06 | 패키지 구조 | naming-guide, architecture-rules 보강 |
| O-07 | 시퀀스 다이어그램 | Use Case 흐름 정확도 향상 |

## 규모별 적용

| 규모 | HLD 검증 | LLD 검증 | 비고 |
|------|---------|---------|------|
| full | H-01~H-07 | L-01~L-09 | 전체 검증 |
| lld-only | 스킵 | L-01~L-09 | LLD만 검증 |
| request-only | 스킵 | 스킵 | 사용자 요청에서 최소 정보 확인 |

## 리포트 형식

```text
⚠️ HLD/LLD 품질 체크 결과
============================================================
{각 항목별 ✅ 통과 / ❌ 누락 + 영향 설명}

선택 항목:
{각 항목별 ⚠️ 없음 + 영향 설명}

진행하시겠습니까? (y/n)
```
