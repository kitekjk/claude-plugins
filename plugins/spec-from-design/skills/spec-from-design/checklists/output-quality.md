# 출력 품질 체크리스트

생성된 Spec의 품질을 평가합니다. spec-reviewer가 사용합니다.

## 평가 항목 (100점)

### ① 구조 완결성 (10점)
- 모든 필수 섹션 존재
- 기본 정보 필드 완전
- source 필드 존재
- ID 규칙 준수

### ② 모호성 (15점)
- "적절한", "충분한", "빠른" 등 모호한 표현 없음
- 수치가 필요한 곳에 수치 사용
- 검증 조건이 테스트 가능한 문장
- 감점: 모호한 표현 1건당 -2점

### ③ 분기 커버리지 (20점)
- 기본 흐름 + 대안 흐름 포함
- 에러 시나리오 포함
- 상태 전이가 있으면 모든 전이 커버
- 감점: 누락 분기 1건당 -3점

### ④ 참조 완결성 (10점)
- 관련 정책 링크 정확 — **참조된 POLICY 파일이 실제 존재해야 함** (필수 FAIL 조건)
- 관련 Spec 링크 정확
- 관련 모델 명시
- 존재하지 않는 ID 참조 없음 — POLICY, Spec ID 모두 실제 파일/섹션과 대응해야 함
- 감점: 깨진 참조 1건당 -3점
- **즉시 FAIL**: 존재하지 않는 POLICY ID를 참조하는 경우

### ⑤ 테스트 도출 가능성 (15점)
- Given-When-Then 형식 준수
- TC-ID 마킹
- 레벨 분류 (Unit/Integration/E2E)
- 검증 조건과 테스트 1:1 매핑
- 감점: 매핑 누락 1건당 -2점

### ⑥ HLD/LLD 추적성 (10점)
- 각 Spec에 source 필드 존재
- HLD KDD → 정책 규칙 전수 매핑
- LLD FR → Use Case 전수 매핑
- LLD FR 검증 기준 → 테스트 전수 매핑
- LLD 미해결 리스크 → 엣지 케이스 테스트 매핑
- 감점: 미추적 항목 1건당 -3점

### ⑦ YAGNI (10점)
- HLD/LLD에 명시되지 않은 기능이 Spec에 추가되지 않음
- 불필요한 추상화 계층이 없음 (e.g., 단일 구현인데 인터페이스 분리)
- 미래 확장을 위한 예비 필드/파라미터가 없음
- 요구사항에 없는 설정 가능성(configurability)이 추가되지 않음
- 감점: 불필요한 요소 1건당 -3점

### ⑧ 스코프 적정성 (10점)
- 하나의 Spec이 단일 관심사(single concern)에 집중
- 독립적으로 구현·테스트·배포 가능한 단위인지 확인
- Spec이 너무 크면 분해 권고 (기본 흐름 + 대안 흐름 합계 15단계 초과 시 경고)
- Spec이 너무 작으면 통합 권고 (단일 CRUD만 포함하고 비즈니스 로직 없음)
- 감점: 스코프 부적절 시 -5점

## 통과 기준

- 파일별 80점 이상
- 전체 평균 80점 이상
- 개별 항목 50% 미만이면 FAIL
- ⑥ 추적성 10점 미만이면 FAIL (request-only 제외)
- ⑦ YAGNI 5점 미만이면 FAIL
- ⑧ 스코프 적정성 5점 미만이면 FAIL
