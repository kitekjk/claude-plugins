# 추적성 체크리스트

HLD/LLD → Spec 변환의 완전성을 검증합니다.

## 추적성 검증 항목

### 1. HLD KDD → 정책 Spec (full 모드)

각 HLD KDD가 최소 1개의 정책 규칙으로 변환되었는지 확인합니다.

```text
KDD-{N} {결정} → POLICY-{DOMAIN}-{번호} {규칙}
```

### 2. LLD FR → Use Case (full, lld-only 모드)

각 LLD FR이 Use Case의 기본 흐름 또는 대안 흐름으로 변환되었는지 확인합니다.

```text
FR-{N} → {PREFIX}-{DOMAIN}-{번호} {기본흐름 Step N | AF-N}
```

### 3. LLD FR 검증 기준 → 테스트 시나리오

각 FR 검증 기준이 최소 1개의 테스트 시나리오로 변환되었는지 확인합니다.

```text
FR-{N} 검증기준 → TC-{DOMAIN}-{번호}-{시나리오번호}
```

### 4. LLD 미해결 리스크 → 엣지 케이스 테스트

각 미해결 리스크가 엣지 케이스 테스트로 변환되었는지 확인합니다.

```text
Risk-{N} {리스크} → TC-{DOMAIN}-{번호}-{시나리오번호}
```

### 5. HLD NFR → NFR 테스트 (full 모드)

주요 NFR 지표가 테스트 시나리오로 변환되었는지 확인합니다.

## 리포트 형식

```text
📋 HLD/LLD → Spec 추적성 검증
============================================================
HLD KDD 커버리지: {N}/{M} ({%})
  {각 KDD 매핑 상태}

LLD FR 커버리지: {N}/{M} ({%})
  {각 FR 매핑 상태}

테스트 커버리지: {N}/{M} ({%})
  {각 검증 기준/리스크 매핑 상태}

전체 커버리지: {%}
============================================================
```
