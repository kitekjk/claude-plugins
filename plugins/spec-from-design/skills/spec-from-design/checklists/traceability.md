# 추적성 체크리스트 (traceability)

> **사용 시점**: spec-traceability-verifier(⑨)가 전수 대조 시 적용. spec-reviewer(⑩)는 형식 평가 시 참조.
> **목적**: HLD/LLD의 각 요소가 Spec에 올바르게 반영되었는지 양방향 추적 검증

---

## 추적 방향

```
HLD/LLD ──→ Spec (순방향: 설계 요소가 Spec에 반영되었는가)
Spec ──→ HLD/LLD (역방향: Spec의 내용이 설계에 근거가 있는가)
```

---

## 1. HLD KDD → Policy 추적

> HLD의 핵심 설계 판단(KDD)이 Policy로 추출되었는지 확인합니다.

### 체크 항목

- [ ] **TR-H-01** HLD에 존재하는 모든 KDD 항목이 Policy 파일로 추출되었는가?
- [ ] **TR-H-02** 각 Policy 파일의 `출처:` 필드가 대응하는 HLD KDD 항목을 참조하는가?
- [ ] **TR-H-03** HLD NFR(비기능 요구사항)이 POLICY-NFR-001 또는 개별 NFR Policy로 추출되었는가?
- [ ] **TR-H-04** HLD 상태 전이 모델이 관련 UC의 `대안 흐름` 또는 상태 Policy 규칙에 반영되었는가?
- [ ] **TR-H-05** HLD System Context의 외부 시스템 의존성이 infra-config.md에 반영되었는가?

### 누락 보고

KDD가 Policy로 추출되지 않은 경우:
```
[추적성 누락: HLD KDD → Policy]
누락된 KDD: {KDD 항목 설명}
위치: HLD 섹션 {번호}
조치: policy-extractor가 해당 KDD를 Policy로 추출해야 합니다.
```

## 2. LLD FR → UC 기본 흐름 추적

> LLD의 기능 요구사항(FR)이 UC Spec의 기본 흐름으로 변환되었는지 확인합니다.

### 체크 항목

- [ ] **TR-L-01** LLD의 모든 FR 항목이 대응하는 UC Spec의 기본 흐름 단계에 포함되었는가?
- [ ] **TR-L-02** 기본 흐름의 각 단계가 특정 LLD FR과 매핑 가능한가? (FR 번호 또는 설명으로 확인)
- [ ] **TR-L-03** LLD에 있는 FR이 Spec에서 누락되지 않았는가? (역방향 확인)
- [ ] **TR-L-04** Spec 기본 흐름에 LLD에 없는 단계가 추가되지 않았는가? (YAGNI 확인)

### 매핑 테이블 (검토자가 작성)

| FR-001 | | | |
| FR-002 | | | |

## 3. FR 검증 기준 → 테스트 시나리오 추적

> LLD의 각 FR 검증 기준이 테스트 시나리오로 변환되었는지 확인합니다.

### 체크 항목

- [ ] **TR-T-01** LLD에 검증 기준이 있는 FR은 대응하는 테스트 시나리오가 존재하는가?
- [ ] **TR-T-02** 각 테스트 시나리오의 Given-When-Then이 LLD 검증 기준에서 도출 가능한가?
- [ ] **TR-T-03** 정상 케이스(positive) 시나리오가 각 FR당 최소 1개 존재하는가?
- [ ] **TR-T-04** 비정상 케이스(negative) 시나리오가 검증 기준에 명시된 에러 조건을 다루는가?

### 매핑 테이블 (검토자가 작성)

| LLD FR 검증 기준 | 대응 TC-ID | 시나리오 유형 |
|----------------|-----------|------------|
| | | positive |
| | | negative |
| | | edge-case |

## 4. 미해결 리스크 → 엣지 케이스 테스트 추적

> LLD의 미해결 리스크가 엣지 케이스 테스트 시나리오로 반영되었는지 확인합니다.

### 체크 항목

- [ ] **TR-R-01** LLD의 모든 미해결 리스크 항목이 식별되었는가?
- [ ] **TR-R-02** 각 미해결 리스크에 대응하는 엣지 케이스 테스트 시나리오가 존재하는가?
- [ ] **TR-R-03** 엣지 케이스 시나리오의 TC-ID가 부여되었는가? (레벨: Integration 또는 E2E)
- [ ] **TR-R-04** 리스크가 현재 테스트로 검증 불가능한 경우, 해당 사유가 주석으로 기록되었는가?

### 매핑 테이블 (검토자가 작성)

|----------------|------|-----------|-----------|
| | High/Mid/Low | | |

## 5. 역방향 추적 (Spec → HLD/LLD)

> Spec의 내용이 HLD/LLD에 근거가 있는지 확인합니다.

### 체크 항목

- [ ] **TR-B-01** 각 Spec의 `출처:` 필드가 실제 HLD/LLD 섹션을 가리키는가?
- [ ] **TR-B-02** Spec의 기본 흐름 단계 중 LLD에 없는 단계가 없는가?
- [ ] **TR-B-03** Spec의 대안 흐름이 LLD 설계 판단 또는 HLD 상태 모델에 근거하는가?
- [ ] **TR-B-04** Spec의 관련 정책이 실제 추출된 Policy 파일과 연결되는가?

## 판정 기준

| 상태 | 조건 |
|------|------|
| **pass** | 모든 TR-H, TR-L, TR-T, TR-R, TR-B 항목 통과 |
| **fail** | TR-H-01, TR-L-01, TR-L-03 중 하나라도 미통과 |
| **warn** | 나머지 항목 일부 미통과 (감점은 output-quality Q6에 반영) |
