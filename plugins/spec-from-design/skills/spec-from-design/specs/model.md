# model 유형 정의

## 목적

엔티티/스키마 레이어의 변경 사항을 기술한다.
DB 테이블 변경, 필드 추가/삭제, 관계 설정, 마이그레이션 계획을 명세한다.

## 생성 조건

- scope-evaluator가 `split-needed` 판정한 usecase Spec의 분해 산출물로만 생성
- usecase-splitter가 분해 시 생성 — 단독 생성 금지 (hardGuardrail)
- 부모 usecase의 모델 변경 범위가 분리 기술이 필요할 정도로 큰 경우

## 필수 섹션

- 기본 정보 (유형, 도메인, 출처, 부모 usecase 참조)
- 개요
- 모델 변경사항 (추가/수정/삭제 엔티티 목록)
- 필드 정의 (각 필드명, 타입, 제약, 설명)
- 관계 (연관 엔티티, 관계 유형, 카디널리티)
- 마이그레이션 (변경 순서, 롤백 방법, 데이터 변환 로직)

## 선택 섹션

없음

## 제약 사항

- 단독 생성 금지 — 반드시 `split-needed` 판정에서 비롯된 분해 산출물이어야 함
- 부모 usecase Spec을 기본 정보에 명시적으로 참조해야 함
- 구현 코드 금지 (DDL pseudocode 허용)
- 1 Spec = 1 파일
