# Model Spec 정의

엔티티/스키마/리포지토리 레이어 변경에 사용합니다.
대규모 Use Case 분해 시 모델 레이어를 **선행 개발**할 때 사용합니다.
**구현 코드를 포함하지 않습니다**.

## 언제 사용하는가

- 대규모 UC를 분해할 때, 엔티티와 리포지토리를 먼저 만들어야 하는 경우
- 독립적인 스키마 변경 (인덱스 추가, 컬럼 추가 등)
- 소규모 UC에서는 usecase Spec 하나에 모델 정보를 포함하므로 별도 model Spec 불필요

## 공통 섹션

기본 정보, 수정 대상 파일, dependsOn, 관련 정책, 관련 Spec, 관련 모델, 검증 조건, 테스트 시나리오

## model 고유 섹션

| 섹션 | 설명 |
|------|------|
| 개요 | 모델 변경의 목적과 배경 |
| 모델 변경 내역 | 신규/수정 엔티티 목록과 변경 사유 |
| 필드 정의 | 각 엔티티의 필드명, 타입, 제약조건, 용도를 자연어로 기술 |
| 관계 정의 | 엔티티 간 관계 (1:N, N:M 등)를 자연어로 기술 |
| 마이그레이션 설명 | 스키마 변경 시 필요한 마이그레이션 설명 (DDL 코드가 아닌 자연어) |

## dependsOn 특성

- model Spec은 보통 **의존성이 없음** (선행 레이어)
- 다른 model Spec과 같은 엔티티를 수정하면 dependsOn 기록
