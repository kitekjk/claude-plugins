# performance 유형 정의

## 목적

성능 개선 작업의 현재 문제, 해결 방안, 목표 지표를 기술한다.
수치 기반 목표를 제시하여 측정 가능한 성능 개선 명세를 작성한다.

## 생성 조건

- LLD에서 명시적으로 `performance` 유형으로 분류한 항목만 생성
- LLD의 "성능 최적화", "응답 시간 개선", "처리량 향상" 명시 항목이 해당됨
- NFR(비기능 요구사항)에서 특정 성능 지표가 명시된 경우

## 필수 섹션

- 기본 정보 (유형, 도메인, 출처, 구현 상태)
- 개요 (성능 개선 목표 요약)
- 현재 문제 (현재 성능 수치, 병목 지점, 문제 원인)
- 해결 방안 (적용할 최적화 기법, 변경 사항)
- 목표 지표 (응답 시간, 처리량, 에러율 등 수치 기반 목표)
- 측정 방법 (성능 측정 방식, 도구, 기준 환경)

## 필수 테스트 시나리오

- 테스트 시나리오 섹션을 필수로 포함한다
- Given-When-Then 형식, TC-ID 부여, 테스트 레벨 분류(Unit/Integration/E2E)
- 유형별 가이드:
  - 기능 동등성 검증: 성능 개선 후에도 기존 기능이 동일하게 동작하는지
  - 목표 지표 달성 여부: 응답시간, 처리량, 메모리 사용량 등 수치 목표 충족 여부
  - 부하 시나리오: 동시 접속, 대용량 데이터 등 극한 조건에서의 동작 검증

## 선택 섹션

없음

## 제약 사항

- LLD 명시 없이 임의 생성 금지
- 목표 지표는 반드시 수치로 명시 (예: "응답 시간 500ms → 100ms 이하")
- 구현 코드 금지 (pseudocode 허용)
- 1 Spec = 1 파일
- 구현 상태 초기값: `not-started`
