# Performance Spec 정의

동작은 동일하지만 성능을 개선하는 변경에 사용합니다.
쿼리 최적화, 캐시 추가, N+1 해결, 배치 크기 조정 등.
**구현 코드를 포함하지 않습니다**.

## 언제 사용하는가

- 쿼리 최적화, 인덱스 추가
- 캐시 레이어 도입
- N+1 문제 해결
- 배치 크기/병렬도 조정
- 응답 시간 개선

## 공통 섹션

기본 정보, 수정 대상 파일, dependsOn, 관련 정책, 관련 Spec, 관련 모델, 검증 조건, 테스트 시나리오

## performance 고유 섹션

| 섹션 | 설명 |
|------|------|
| 개요 | 성능 개선의 목적과 배경 |
| 현재 성능 문제 | 현재 성능 수치와 문제점 |
| 개선 방안 | 적용할 최적화 기법을 자연어로 설명 |
| 목표 수치 | 개선 후 달성할 성능 목표 (P50, P99, TPS 등) |
| 측정 방법 | 성능 개선을 검증할 측정 방법 |

## 핵심 제약

- **동작 변경 없음**: 최적화 전후 동일한 입력에 동일한 출력이 보장되어야 함
- 기존 테스트가 모두 통과해야 함
- 동작이 바뀌면 usecase Spec을 사용
