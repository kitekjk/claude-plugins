# policy 유형 정의

## 목적

도메인 레벨에서 재사용 가능한 설계 규칙과 제약조건을 기술한다.
여러 UseCase에 공통 적용되는 비즈니스 정책 및 아키텍처 결정을 명세한다.

## 생성 조건

- policy-extractor가 HLD KDD 또는 LLD 설계 판단에서 추출한 항목만 생성
- 두 개 이상의 UseCase에 적용되는 공통 규칙인 경우
- HLD NFR에서 비기능 요구사항으로 명시된 경우 (POLICY-NFR-NNN으로 생성)
- HLD 상태 전이 모델에서 상태 규칙이 명시된 경우

## 필수 섹션

- 기본 정보 (유형, 도메인, 출처, 버전)
- 규칙 (정책 규칙 목록, 각 규칙의 근거)
- 제약조건 (규칙 위반 시 결과, 예외 허용 조건)
- 적용 도메인 (이 정책이 적용되는 도메인/컴포넌트 목록)

## 선택 섹션

없음

## 제약 사항

- policy-extractor 외 에이전트의 직접 생성 금지
- UseCase 특화 규칙은 Policy가 아닌 해당 Spec의 검증 조건에 기술
- 기존 Policy와 중복 시 새 파일 생성 대신 적용 도메인 항목 추가
- 파일명 형식: `POLICY-{DOMAIN}-{NNN}.md` 또는 `POLICY-NFR-{NNN}.md`
- 구현 코드 금지
