# refactoring 유형 정의

## 목적

기능 변경 없이 코드 구조를 개선하는 작업을 기술한다.
Before/After 구조를 명확히 제시하여 변경 의도와 영향 범위를 명세한다.

## 생성 조건

- LLD에서 명시적으로 `refactoring` 유형으로 분류한 항목만 생성
- LLD의 "리팩토링", "구조 개선", "기술 부채 해소" 명시 항목이 해당됨
- 기능 요구사항(FR)이 없는 순수 구조 변경인 경우

## 필수 섹션

- 기본 정보 (유형, 도메인, 출처, 구현 상태)
- 개요 (리팩토링 목표 요약)
- Before 구조 (현재 코드 구조, 문제점)
- After 구조 (변경 후 코드 구조, 개선 내용)
- 변경 사유 (리팩토링이 필요한 근거, 기술 부채 맥락)
- 영향 범위 (변경되는 파일, 연동 시스템, 하위 호환성 여부)

## 선택 섹션

없음

## 제약 사항

- LLD 명시 없이 임의 생성 금지
- 기능 변경이 포함되면 refactoring이 아닌 usecase로 분류
- 구현 코드 금지 (pseudocode 허용)
- 1 Spec = 1 파일
- 구현 상태 초기값: `not-started`
