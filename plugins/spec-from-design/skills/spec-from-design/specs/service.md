# service 유형 정의

## 목적

애플리케이션/도메인 서비스 레이어의 책임과 메서드를 기술한다.
서비스 클래스의 역할 경계, 입출력, 에러 처리 방침을 명세한다.

## 생성 조건

- scope-evaluator가 `split-needed` 판정한 usecase Spec의 분해 산출물로만 생성
- usecase-splitter가 분해 시 생성 — 단독 생성 금지 (hardGuardrail)
- 부모 usecase의 서비스 레이어 범위가 분리 기술이 필요할 정도로 큰 경우

## 필수 섹션

- 기본 정보 (유형, 도메인, 출처, 부모 usecase 참조, 구현 상태)
- 개요
- 서비스 책임 (단일 책임 원칙에 따른 역할 정의)
- 메서드 (메서드명, 역할 설명 목록)
- 입출력 (각 메서드의 입력 파라미터, 반환 타입, 설명)
- 에러 처리 (발생 가능 예외, 처리 방침, 반환 코드)

## 선택 섹션

없음

## 제약 사항

- 단독 생성 금지 — 반드시 `split-needed` 판정에서 비롯된 분해 산출물이어야 함
- 부모 usecase Spec을 기본 정보에 명시적으로 참조해야 함
- 구현 코드 금지 (pseudocode 허용)
- 메서드 시그니처는 언어 코드가 아닌 자연어로 기술
- 1 Spec = 1 파일
- 구현 상태 초기값: `not-started`
