# Service Spec 정의

도메인 로직 레이어 변경에 사용합니다.
대규모 Use Case 분해 시 서비스 레이어를 **중간 개발**할 때 사용합니다.
**구현 코드를 포함하지 않습니다**.

## 언제 사용하는가

- 대규모 UC를 분해할 때, model 완성 후 도메인 로직을 구현하는 경우
- 복잡한 도메인 로직이 독립적으로 테스트 가능한 경우
- 소규모 UC에서는 usecase Spec 하나에 서비스 정보를 포함하므로 별도 service Spec 불필요

## 공통 섹션

기본 정보, 수정 대상 파일, dependsOn, 관련 정책, 관련 Spec, 관련 모델, 검증 조건, 테스트 시나리오

## service 고유 섹션

| 섹션 | 설명 |
|------|------|
| 개요 | 서비스의 책임과 역할 |
| 서비스 책임 | 이 서비스가 담당하는 비즈니스 규칙 목록 |
| 메서드 설명 | 추가/변경할 메서드의 역할, 입력, 출력을 자연어로 기술 (코드 아님) |
| 입출력 정의 | 각 메서드의 파라미터와 반환값을 자연어로 기술 |
| 에러 처리 | 발생 가능한 에러와 처리 방식 |

## dependsOn 특성

- 보통 **model Spec에 의존** (엔티티를 사용하므로)
- usecase Spec이 이 service Spec에 의존
- 순서: model → **service** → usecase
