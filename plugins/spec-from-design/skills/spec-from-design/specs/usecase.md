# usecase 유형 정의

## 목적

외부 진입점을 통해 시작되는 하나의 기능 단위를 기술한다.
진입점 유형에 해당하는 각 구현 클래스를 코드 생성 가능한 수준으로 명세한다.

## 생성 조건

- 진입점 유형에 해당하는 구현 클래스가 식별되었을 때
- 기본 유형 — 별도 분류 판정 없이 자동 할당
- usecase-identifier가 UC 식별 목록에 포함한 모든 클래스에 대해 생성

## 필수 섹션

- 기본 정보 (유형, 도메인, 출처, 진입점 유형, 구현 클래스)
- 개요
- 수정 대상 파일
- 의존성 (dependsOn)
- 기본 흐름
- 검증 조건
- 테스트 시나리오

## 선택 섹션

- 대안 흐름 (분기가 있는 경우)
- 관련 정책 (적용되는 Policy가 있는 경우)

## 제약 사항

- 구현 코드 금지 (pseudocode 허용)
- 1 Spec = 1 파일 (규모와 무관하게 하나의 파일로 작성)
- LLD 내용(모델 변경, 도메인 로직)에 의한 유형 변경 금지
- scope-evaluator 판정이 `split-needed`인 경우에만 model/service로 분해 가능
