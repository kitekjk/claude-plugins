# 구현 완료 체크리스트

## 필수 검증 항목

### I-01: 프로젝트 초기화 (Pass/Fail)
- [ ] 빈 디렉토리에서 시작했는가
- [ ] Spec 문서가 `docs/specs/`에 복사되었는가
- [ ] CLAUDE.md가 루트에 복사되었는가

### I-02: 빌드 성공 (Pass/Fail)
- [ ] 빌드 도구 실행 성공 (예: `./gradlew build`)
- [ ] 컴파일 에러 0건
- [ ] 경고 확인 (치명적 경고 없음)

### I-03: 테스트 통과 (Pass/Fail)
- [ ] 전체 테스트 실행 성공 (예: `./gradlew test`)
- [ ] 실패 테스트 0건
- [ ] 스킵 테스트 확인 (의도적 스킵만 허용)

### I-04: TC-ID 커버리지 (Pass/Fail)
- [ ] Spec의 모든 TC-ID가 테스트 코드에 존재
- [ ] @Tag 또는 동등한 마킹으로 TC-ID가 연결됨
- [ ] 누락 TC-ID 목록 보고

### I-05: 아키텍처 준수 (Pass/Fail)
- [ ] architecture-rules.md의 레이어 분리 준수
- [ ] 도메인 레이어에 프레임워크 의존성 없음
- [ ] 의존성 방향 규칙 준수

### I-06: 네이밍 준수 (Pass/Fail)
- [ ] naming-guide.md의 패키지 네이밍 준수
- [ ] naming-guide.md의 클래스 네이밍 준수
- [ ] naming-guide.md의 API 경로 네이밍 준수

## 재시도 규칙

- I-02 또는 I-03 실패 시 수정 후 재검증 (최대 3회)
- I-04 실패 시 누락 테스트 추가 후 재검증
- I-05, I-06 실패 시 수정 후 재검증
- 3회 재시도 후에도 실패 시 실패 리포트 반환
